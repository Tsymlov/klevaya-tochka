# КлёваяТочка: План Разработки

## 1. Продукт

`КлёваяТочка` — публичный сайт с интерактивной картой рыболовных мест.

Ценность MVP:
- любой посетитель видит карту и метки;
- авторизованный пользователь может добавить свою рыболовную точку и описание;
- новые, измененные и удаленные метки появляются у всех пользователей в реальном времени.

## 2. Правила MVP

- Сайт открыт для всех на чтение.
- Метки полностью публичны.
- Тип контента: только рыболовные места.
- Данные метки: координата на карте и текстовое описание.
- Обновление карты: в реальном времени.
- Владеть, изменять и удалять метку может только ее автор.

## 3. Функциональный Объем MVP

### Пользовательские сценарии

1. Посетитель открывает главную страницу и видит карту с публичными точками.
2. Пользователь проходит легкую авторизацию.
3. Пользователь ставит метку на карте.
4. Пользователь добавляет краткое описание точки.
5. Пользователь видит новые точки других пользователей без перезагрузки страницы.
6. Пользователь может отредактировать или удалить только свои метки.

### Что не входит в MVP

- фото;
- виды рыбы;
- погода;
- личные/скрытые точки;
- друзья, подписки и чаты;
- комментарии и лайки;
- мобильное приложение.

## 4. Технический Стек

### Backend

- Python 3.12+;
- Django 5;
- Django REST Framework;
- Django Channels;
- PostgreSQL + PostGIS;
- Redis.

### Frontend

- React + TypeScript + Vite;
- MapLibre GL JS.

### Инфраструктура

- Docker Compose для локальной разработки;
- Caddy или Nginx для production;
- Git + GitHub для исходного кода.

## 5. Архитектура

### Репозиторий

- `backend/` — Django, API, WebSocket, админка, безопасность;
- `frontend/` — React-приложение с картой;
- `docs/` — план, решения, эксплуатационные заметки;
- `docker-compose.yml` — локальная среда.

### Основные сущности

#### FishingSpot

- `id`
- `owner`
- `location` (`PointField`)
- `description`
- `created_at`
- `updated_at`

### API MVP

- `GET /api/health/`
- `GET /api/spots/`
- `POST /api/spots/`
- `GET /api/spots/{id}/`
- `PATCH /api/spots/{id}/`
- `DELETE /api/spots/{id}/`
- `POST /api/auth/register/`
- `POST /api/auth/login/`
- `POST /api/auth/logout/`
- `GET /api/auth/me/`
- `WS /ws/spots/`

### Real-Time

- WebSocket-канал передает события `created`, `updated`, `deleted`;
- фронтенд обновляет карту без перезагрузки;
- события транслируются через Redis channel layer.

## 6. Безопасность

### Базовые правила

- чтение разрешено всем;
- запись только авторизованным пользователям;
- изменение и удаление только владельцу метки;
- production только через HTTPS;
- секреты и ключи хранятся только в переменных окружения.

### Обязательные меры для MVP

- сессионная авторизация Django;
- `CSRF`-защита;
- `HttpOnly` для session cookie;
- `SameSite=Lax` для cookie;
- валидация координат и текста на сервере;
- запрет рендера пользовательского текста как HTML;
- rate limit на регистрацию, вход и создание меток;
- ограничение частоты публикации точек;
- логирование попыток входа, ошибок авторизации и массового создания точек;
- CORS только для доверенного origin фронтенда;
- проверка origin для WebSocket;
- резервное копирование PostgreSQL.

### Администрирование

- Django Admin для модерации;
- возможность скрыть спам-метку;
- возможность заблокировать пользователя.

## 7. Этапы Реализации

### Этап 1. Каркас проекта

- инициализировать структуру backend/frontend;
- добавить Docker Compose;
- настроить Django settings;
- подготовить React/Vite-каркас;
- зафиксировать `.env.example`, `README`, `.gitignore`.

### Этап 2. Backend MVP

- модель `FishingSpot`;
- CRUD API;
- объектные права;
- фильтрация по `bbox`;
- auth endpoints;
- админка.

### Этап 3. Real-Time

- Channels routing;
- Redis channel layer;
- отправка событий о создании, редактировании и удалении меток;
- обновление карты на фронтенде через WebSocket.

### Этап 4. Frontend MVP

- главная страница с картой;
- загрузка точек из API;
- попап и карточка точки;
- форма добавления/редактирования;
- отображение своих и чужих меток;
- реакция на real-time события.

### Этап 5. Доводка и Деплой

- rate limiting;
- базовая антиспам-защита;
- мониторинг и логи;
- production-конфигурация;
- инструкции по деплою и публикации на GitHub.

## 8. Definition of Done Для MVP

- карта открывается без авторизации;
- публичные точки загружаются по области карты;
- авторизованный пользователь может создать точку;
- пользователь может редактировать и удалять только свои точки;
- другие пользователи видят изменения в реальном времени;
- проект поднимается локально через Docker Compose;
- базовые меры безопасности включены;
- репозиторий готов к публикации на GitHub.
